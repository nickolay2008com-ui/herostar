# HeroStar ✦

Интерактивная карта личности по данным рождения. Приложение считает астрономические положения отдельно, а GPT используется только для человеческого синтеза: уникальное проявление, контраст с противоположным типом, ловушка и практический ключ.

## Production

- Railway service: `herostar`
- Публичный адрес: `https://herostar.up.railway.app`
- Панель администратора: `https://herostar.up.railway.app/admin`
- PostgreSQL: подключён отдельным Railway service
- Ветка автодеплоя: `main`
- Healthcheck: `https://herostar.up.railway.app/health`
- Подробная фиксация инфраструктуры: [`docs/railway-production.md`](docs/railway-production.md)

## Что уже реализовано

- ввод имени, даты, времени и места рождения;
- автоматическое определение координат и исторического часового пояса;
- расчёт Солнца, Луны, планет, Северного узла, Асцендента, МС и основных аспектов;
- равнодомная система домов в первой версии;
- 11 интерактивных «сокровищ» вместо длинного отчёта;
- анти-Барнум механика: точное положение → проявление → противоположный тип → ловушка → действие;
- интерактивное колесо, фильтры «Ядро / Энергия / Любовь / Действие / Рост / Маршрут»;
- GPT-синтез через OpenAI Responses API со Structured Outputs;
- локальный содержательный fallback, если `OPENAI_API_KEY` не задан;
- консультации по готовой карте с сохранением всей истории в PostgreSQL;
- Telegram Login с серверной проверкой подписи и автоматическим определением username бота по токену;
- YooKassa redirect-платёж и webhook с повторной проверкой платежа через API;
- админ-панель: метрики, воронка, события, разборы, диалоги и оплаты;
- PostgreSQL на Railway, с временным in-memory fallback для локального запуска;
- серверная редакция закрытых карточек, а не декоративное размытие готового текста.

## Архитектура

```text
public/             основной интерфейс и админ-панель без сборщика
src/astro.js        астрономический расчёт и аспекты
src/narrative.js    локальная интерпретация и продуктовая механика
src/ai.js           GPT-синтез и консультант
src/auth.js         Telegram login, сессия и доступ администратора
src/payments.js     YooKassa
src/store.js        PostgreSQL / in-memory, диалоги и аналитика
server.js           HTTP API, доступы и admin API
```

Расчёт планет основан на `astronomy-engine`. Дома первой версии равнодомные: это снижает риск хрупкой нативной сборки на Railway. Архитектура позволяет заменить модуль углов и домов на Swiss Ephemeris без переделки интерфейса и GPT-слоя.

## Локальный запуск

```bash
cp .env.example .env
npm install
npm start
```

Откройте `http://localhost:3000`.

## Развёртывание на Railway

Production уже создан. Для повторного развёртывания:

1. Подключите GitHub-репозиторий к Railway service.
2. Добавьте PostgreSQL service. `DATABASE_URL` появится автоматически.
3. Добавьте переменные из `.env.example`.
4. Railway определит Node.js и запустит `node server.js`. В `railway.json` настроен healthcheck `/health`.

### Минимальные переменные

```text
APP_URL=https://herostar.up.railway.app
SESSION_SECRET=длинная-случайная-строка
OPENAI_API_KEY=...
OPENAI_MODEL=gpt-5-mini
```

## Telegram Login

Создайте бота через BotFather, затем:

1. выполните `/setdomain` и укажите `herostar.up.railway.app`;
2. добавьте в Railway:

```text
TELEGRAM_BOT_TOKEN=...
TELEGRAM_BOT_USERNAME=имя_бота_без_@
TELEGRAM_ADMIN_IDS=ваш_telegram_id
```

`TELEGRAM_BOT_USERNAME` можно оставить пустым: HeroStar вызывает Telegram `getMe` и берёт настоящий username из токена. Это устраняет ошибку «Бот не существует», когда в Railway случайно записан чужой username или добавлен символ `@`.

Несколько администраторов указываются через запятую:

```text
TELEGRAM_ADMIN_IDS=123456789,987654321
```

Callback:

```text
https://herostar.up.railway.app/auth/telegram/callback
```

Подпись, время авторизации и HMAC проверяются на сервере.

## Панель администратора и статистика

Панель открывается по адресу:

```text
https://herostar.up.railway.app/admin
```

Доступ разрешён только Telegram-аккаунтам из `TELEGRAM_ADMIN_IDS`.

В PostgreSQL сохраняются:

- полная карта и полный AI-разбор;
- вопросы пользователя и ответы консультанта;
- посещения, начало формы, создание и просмотр карты;
- открытия карточек и разделов;
- Telegram login и сохранение карты;
- начало и успешное завершение оплаты;
- случайный visitor ID браузера без хранения сырого IP.

В панели можно открыть конкретный разбор и увидеть данные рождения, полный текст, весь диалог, оплаты и хронологию событий.

## YooKassa

Добавьте:

```text
YOOKASSA_SHOP_ID=...
YOOKASSA_SECRET_KEY=...
FULL_MAP_PRICE=990.00
```

Webhook для события `payment.succeeded`:

```text
https://herostar.up.railway.app/api/payments/webhook
```

Приложение не доверяет входящему телу webhook и повторно запрашивает объект платежа у ЮKassa. Доступ открывается только после статуса `succeeded` и `paid=true`.

## Важные продуктовые решения

- **Расчёт и интерпретация разделены.** GPT не определяет знаки и дома.
- **Не продаётся обещание судьбы.** Продаётся ясный образ, сравнение и практический маршрут.
- **Не нужна регистрационная стена до ценности.** Первые карточки открываются сразу, Telegram нужен для сохранения, консультаций и оплаты.
- **Аналитика связана с разбором.** Не отдельная свалка счётчиков, а путь конкретного человека от входа до диалога и оплаты.
